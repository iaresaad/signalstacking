> This is the ONE input the public web can't provide: who at a prospect is ALREADY
> using Trumpet (free tier / trial), and how actively. It powers the highest-converting
> line in an email ("AE Sam created 30 rooms, shared 83").
>
> The skill works fully WITHOUT this. Default is signals-only with peer proof
> (Gong, Cognism, etc.). Only fill the REAL file (trumpet-usage-data.md, which is
> gitignored) when you have data from Trumpet's product/CRM (e.g. sign-ups by domain).
>
> ⚠️ INTEGRITY RULE: the skill ONLY uses usage stats that appear in the file. It must
> never invent rep names, room counts, or engagement numbers. No data = no usage line.
